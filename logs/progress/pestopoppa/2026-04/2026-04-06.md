## Session 2 — Harness Engineering Patterns Upstream (intake-271–274)

### Context

Research intake in epyc-root processed 4 entries about coding agent harness engineering (HumanLayer blog, ETH Zurich AGENTS.md evaluation, Chroma context rot, Complexity Trap). Deep-dive analysis identified 6 portable patterns and 4 anti-patterns that generalize beyond EPYC to any Claude Code project.

### Completed

**New handoff**: `handoffs/active/harness-engineering-patterns.md`
- 6 patterns: CLI > MCP for known services, agent file sizing (≤400 words toolchain-only), sub-agent context isolation (not role specialization), PostStop verification hook, two-layer context compression, instruction budget awareness
- 4 anti-patterns: pre-emptive harness design, MCP hoarding, micro-managed tool access, full test suites at session end
- 5 outstanding tasks: skill templates, hook examples, CLI wrapper docs, decision matrix

**Shared standards update**: `agents/shared/ENGINEERING_STANDARDS.md`
- Added "Instruction Budget Policy" section: essential-toolchain vs nice-to-have classification, ≤400 word target, 20% overhead ceiling

**Shared workflow update**: `agents/shared/WORKFLOWS.md`
- Added "Harness Change Evaluation" workflow: measure baseline → apply change → compare quality/cost/instruction_ratio → accept only if proportionate
