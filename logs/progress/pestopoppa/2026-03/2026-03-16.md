## Upstream Contribution Pipeline — COMPLETE

Implemented a full instance-to-archetype contribution pipeline so governance improvements invented in live root repos can be proposed back as PRs to root-archetype.

### Changes

| File | Action | Purpose |
|------|--------|---------|
| `init-project.sh` | Modified | Writes `.archetype-manifest.json` into new instances, adds to `.gitignore`, copies upstream scripts/skills |
| `scripts/upstream/retroactive-manifest.sh` | Created | Generates manifest for instances created before this feature |
| `scripts/upstream/distill.sh` | Created | Core engine: reverse-templates, contamination checks, staging, handoff generation |
| `scripts/upstream/submit-pr.sh` | Created | Creates branch in archetype, copies staged files, runs validators, pushes, opens PR |
| `.claude/commands/upstream.md` | Created | `/upstream` slash command for Claude Code |
| `.claude/skills/upstream/SKILL.md` | Created | Skill definition for richer integration |

### Pipeline Flow

1. Instance has `.archetype-manifest.json` (written at init or via `retroactive-manifest.sh`)
2. `distill.sh` diffs portable paths, reverse-templates project values to `{{PLACEHOLDERS}}`, blocks contaminated files, stages clean files, generates handoff doc
3. `submit-pr.sh` creates branch in archetype, copies staged files, runs validators, commits, pushes, opens GitHub PR with handoff as body
4. Maintainer reviews handoff in `handoffs/active/` independently

### Why

root-archetype is a factory — after `init-project.sh`, instances are independent. Users frequently invent new governance tools while working in live root repos but had no way to propose them back. This closes the loop while protecting archetype project-agnosticism via reverse-templating and contamination checks.
