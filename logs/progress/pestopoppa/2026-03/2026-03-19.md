## Swarm Observability Module — COMPLETE

### Changes

| File | Action | Purpose |
|------|--------|---------|
| `swarm/metrics.py` | Created (~230 LOC) | `SwarmMetrics` class: read-only metrics exporter |
| `swarm/test_metrics.py` | Created (14 tests) | Full test coverage for all metrics methods |

### API

| Method | What it tracks |
|--------|---------------|
| `queue_counts()` | Work item status distribution |
| `work_latency()` | p50/p95/p99 for wait, exec, total time |
| `budget_burn_rate(window)` | Per-agent spend rate, breakdown by action type |
| `agent_activity()` | Per-agent work counts, active status |
| `info_gain_accuracy()` | Predicted vs actual information gain (MAE) |
| `lock_status()` | Active/expired locks, hold duration |
| `snapshot()` | All metrics combined |
| `emit_to_audit_log()` | JSONL entry compatible with `agent_log.sh` |

### Why

Issue #2's last gap was observability. The coordinator had raw data (timestamps, budget ledger) but no aggregation or export. Now nightshift, cron, or manual invocation can produce structured metrics snapshots compatible with the existing audit trail.
