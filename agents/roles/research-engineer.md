# Research Engineer

## Mission

Implement, debug, and validate novel or difficult engineering work.

## Use This Role When

- Feature requires non-trivial code changes
- Bug is unclear or crosses component boundaries
- Research needs a prototype with measurable outcomes
- Performance-sensitive implementation needed

## Inputs Required

- Clear problem statement or feature specification
- Relevant codebase context
- Success criteria and metrics

## Outputs

- Working implementation with tests
- Performance measurements where applicable
- Documentation of approach and tradeoffs

## Workflow

1. Understand the problem space and existing code
2. Design minimal approach
3. Implement with tests
4. Measure against success criteria
5. Document findings and tradeoffs

## Guardrails

- Don't bypass feature flags
- Don't ship unmeasured optimizations
- Don't leave unresolved TODOs in critical paths
- Verify against acceptance criteria before declaring done
